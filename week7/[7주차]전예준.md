### 스프링 어노테이션

자바 소스 코드에 추가하여 사용할 수 있는 메타데이터의 일종. 설정 정보를 XML 파일이 아닌 자바 코드 내에 직접 표기하여, 스프링 컨테이너가 읽고 자동으로 애플리케이션을 구성하도록 지시하는 역할

-@Bean: 해당 메서드가 반환하는 객체를 Bean으로 수동 등록할 때 사용(수동 등록 방식)
-@Component: 컴포넌트 스캔의 대상이 되는 클래스에 붙이며, 스프링 컨테이너가 이 클래스를 자동으로 찾아 Bean으로 등록(자동 등록 방식)
-@Controller: 스프링 컨테이너에게 이 클래스를 Bean으로 등록
-@Service: 스프링에서 비즈니스 로직을 처리하는 서비스 계층의 클래스에 사용
-@Repository: 스프링에서 데이터베이스관련 로직을 처리하는 계층에 붙이는 어노테이션
-@RequestMapping: 특정 URL경로와 HTTP 메서드를 클래스 또는 메서드에 매핑
-@Get/Post/Put/DeleteMapping: HTTP GET/POST/PUT/DELETE 요청을 매핑


### 스프링 컨테이너

애플리케이션의 필요한 객체들을 대신 생성하고, 관리하며, 서로 연결해주는 객체 관리 시스템
1. 객체의 생성: 개발자는 @Service, @Controller 같은 어노테이션으로 표시 -> 애플리케이션이 시작될 때, 스프링 컨테이너가 이 클래스들을 미리 스캔하여 객체로 생성(빈(Bean))
2. 의존성 주입: OrderController가 OrderService를 필요로 할 때, 개발자는 @Autowired -> 컨테이너는 OrderController 객체를 만들 때, 자신이 이미 만들어 둔 OrderService 객체를 자동으로 찾아서 연결
