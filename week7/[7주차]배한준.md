### 스프링 어노테이션이란

스프링 어노테이션은 스프링 프레임워크가 객체를 생성하거나 의존성을 주입하고 웹 요청을 처리하도록 표시하는 데 사용되는 특별한 주석이다. 코드에 어노테이션을 붙여두면 스프링이 이를 읽어 필요한 빈을 등록하거나 연결해 주며, 개발자는 객체 생성이나 관리 흐름을 직접 작성하지 않아도 되어 생산성이 향상된다.

### 스프링 어노테이션의 종류

스프링 어노테이션의 종류는 크게 역할에 따라 구분된다. @Component, @Service, @Repository, @Controller처럼 특정 클래스를 스프링 빈으로 등록하는 컴포넌트 계열이 있으며, @Autowired나 @Qualifier처럼 의존성 주입을 수행하는 어노테이션이 존재한다. 또한 @RequestMapping, @GetMapping, @PostMapping 등은 HTTP 요청을 처리하기 위해 사용되고, @Configuration, @Bean은 설정 및 빈 등록을 위한 어노테이션이다. 이들은 명확한 역할을 지니며 스프링이 동작할 때 핵심적인 정보를 제공한다.

### 스프링 컨테이너란

스프링 컨테이너는 애플리케이션의 객체를 생성하고 관리하는 실행 환경이다. 컨테이너는 어노테이션 또는 설정 파일을 기반으로 필요한 객체를 빈으로 등록해두고 필요할 때 주입하여 사용하도록 한다. 이를 통해 객체 생성 방식이 일관되게 유지되고 코드 간 결합도가 낮아지며 유지보수가 쉬워진다. 개발자는 컨테이너가 생성해주는 빈을 활용해 로직 구현에 집중할 수 있다.
